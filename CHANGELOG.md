# Changelog

## 現狀總覽

**狀態：** 穩定（分類邏輯、啟動器穩定性、UI 優化全數完成）
**啟動方式：** `台股公告監控雷達.bat` → `http://localhost:7853`

### 既有功能
- **全自動監控**：支援 Windows/macOS/Linux 自動找 Chrome 跑爬蟲。
- **Resend 寄信**：美觀 HTML 卡片，且具備網頁版設定教學彈窗。
- **個人化 UI**：緊湊版 Header、品牌標誌強化、Created by CTH 署名。
- **智能過濾**：24H 回溯、跨月自動補抓、股票名稱自動同步。
- **系統診斷**：支援 `npm run check` 環境自我檢查。

## 更新記錄

### 2026-03-07 中午 (V1.2 穩定分類版)

#### 新增
- **一鍵清除歷史紀錄**：在「05 / 通知歷史」新增「🧹 清除紀錄」按鈕，方便使用者在調整分類規則後清空舊資料重新爬取。
- **Git 版本管理**：正式初始化專案 Git 倉庫並上傳至 GitHub，設定 `.gitignore` 確保 API Key 與資料庫安全不外洩。
- **目錄結構保護**：透過 `.gitkeep` 確保 GitHub 上的 `data/` 資料夾結構完整，同時不影響本地開發。

#### 修改
- **分類邏輯重大革新 (扁平化)**：
  - 捨棄複雜的兩段式過濾，改為直覺的「單次掃描」結構。
  - **優化董監事異動 (board)**：加入二次確認邏輯，標題必須包含「辭職、新任、改選」等變動詞彙才會標記，徹底解決「董事會通過一般議案」被誤判為人事異動的問題。
  - **優化合併/收購 (merger)**：若標題對應到財報性質，自動排除併購標籤，避免「合併財務報告」造成雜訊。
- **關鍵字精確化**：將行業用語如 `委任` 改為 `委任人員` 等更長詞彙（現已開放讓使用者在 `src/crawler.js` 自由調整）。

#### 修復
- **解決標籤顯示 ID 問題**：修正資料庫與前端名稱映射邏輯，確保自定義標籤能正確顯示中文。
