# Changelog

## 現狀總覽

**狀態：** 穩定（分類邏輯、啟動器穩定性、UI 優化全數完成）
**啟動方式：** `台股公告監控雷達.bat`（薄殼 2 行，呼叫 `台股公告監控雷達.ps1`）→ `http://localhost:7853`
**啟動器架構：** 薄 BAT + 台股公告監控雷達.ps1（UTF-8 BOM），所有邏輯與中文訊息集中於 PS1，無中文亂碼問題

### 既有功能
- **全自動監控**：支援 Windows/macOS/Linux 自動找 Chrome 跑爬蟲。
- **Resend 寄信**：美觀 HTML 卡片，且具備網頁版設定教學彈窗。
- **個人化 UI**：緊湊版 Header、品牌標誌強化、Created by CTH 署名。
- **智能過濾**：24H 回溯、跨月自動補抓、股票名稱自動同步。
- **系統診斷**：支援 `npm run check` 環境自我檢查。

### 2026-03-09 (V1.2.6 啟動器元件自動安裝)

#### 新增
- **Node.js 自動安裝**：啟動器偵測到未安裝 Node.js 時，詢問使用者後自動安裝：優先使用 `winget`，無 winget 則用 PowerShell 從 nodejs.org 下載最新 LTS MSI 靜默安裝。安裝後提示重開視窗生效。
- **Chrome 自動安裝**：啟動器新增 Chrome 完整偵測流程：`.env CHROME_PATH` → Windows Registry → 三個常見路徑，都找不到時詢問使用者後自動安裝（`winget` 或 PowerShell 下載 Google 官方安裝程式靜默執行）。找到的路徑自動寫入 `.env`，下次啟動不需重新偵測。
- **手動選擇 Chrome 路徑**：若使用者拒絕安裝但 Chrome 已裝在非標準位置，可透過 GUI 檔案選擇視窗手動指定 `chrome.exe`，路徑同樣寫入 `.env`。

#### 修改
- **啟動器檢查流程擴充**：由原本的 3 步驟擴充為 4 步驟（Node.js → Chrome → Port → npm 套件），全程不需使用者手動開網頁或操作安裝程式。
- **預設值統一**：所有安裝詢問改為「直接按 Enter 代表同意」，降低使用者操作門檻。
