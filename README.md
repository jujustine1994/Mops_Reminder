## 🌟 核心功能
- **全自動監控**：支援 1-30 天回溯範圍，不漏掉任何重要公告。
- **多重標籤分類**：自動辨識財報、股利、重要會議、報導相關等重要訊息（詳見下方分類規則）。
- **視覺化日誌**：直接在網頁上看 Log 進度，不需要看複雜的程式碼。
- **專業 Email 通知**：自動將最新公告寄送到您的信箱，支援行動裝置閱讀。
- **一鍵啟動**：提供 Windows 專用 `.bat` 啟動檔，麻瓜也能輕鬆使用。

## 🏷️ 公告自動分類規則
系統會根據公告標題自動打上標籤，方便快速過濾重點資訊：

| 標籤名稱 | 包含關鍵字 (部分列出) |
| :--- | :--- |
| **重要會議** | 股東會、董事會、召開、決議 |
| **法說會** | 法說會、法人說明會、業績說明會、說明會 |
| **財務報告** | 財報、季報、年報、損益、獲利、盈餘、自結、營收 |
| **股利公告** | 股利、除權、除息、配息、配股、現金、股票 |
| **處分資產** | 處分、取得、資產、不動產、廠房、土地、工程、機器、設備 |
| **股本變動** | 增資、減資、私募、庫藏股、可轉債、發行新股、現增 |
| **報導相關** | 澄清、媒體、報導、報載、新聞、新聞稿、雜誌 |
| **董監事異動** | 辭職、卸任、委任、派任、解任、任期屆滿 |
| **合併/收購** | 合併、收購、分割、購併、併購 |
| **其他重大訊息** | 未符合上述關鍵字之所有其他重大公告 |

- **Notification**: Resend API
- **UI**: Vanilla JS + CSS (Flexbox Flow)

## 📋 股票代號對照表 (stock_ref.db)

專案內含 `data/stock_ref.db`，儲存台股代號與中文名稱的對照表，供顯示與通知用。

| 項目 | 說明 |
| :--- | :--- |
| **收錄範圍** | 上市、上櫃、興櫃掛牌公司 |
| **最後維護** | 2026-03-26（由 CTH 人工維護更新） |
| **自動學習** | 新增監控標的時，若查不到名稱，系統會自動透過爬蟲補入 |

### 使用者自行新增/刪除監控標的
在網頁介面的「監控股票」區塊，直接輸入股票代號按「＋ 新增監控」即可。系統會自動查詢並顯示中文公司名稱（優先從 stock_ref.db 查，查不到才啟動爬蟲）。刪除監控標的按列表右側的「×」即可。

> **注意**：`stock_ref.db` 為公開清單，不含任何個人設定。您的 Email、排程等個人資料存於 `data/mops.db`（不上傳至 GitHub）。

## 📂 目錄結構
- `src/`: 核心程式碼 (爬蟲、資料庫、郵件發送、伺服器)
- `public/`: 網頁端介面
- `data/stock_ref.db`: 股票代號↔中文名稱對照表（隨專案發布）
- `data/mops.db`: 個人設定與通知歷史（本機自動建立，不上傳）
- `台股公告監控雷達.bat`: 薄殼啟動器（呼叫 launcher.ps1）
- `launcher.ps1`: 環境檢查、首次安裝說明、啟動邏輯
- `.env`: 環境變數設定 (API Key 等)

## 🚀 Release 流程
確認目前版本穩定後，將專案資料夾打包複製一份到 Release 資料夾（例如：`Mops_Radar_Release_v1.2.5/`）作為穩定版封存。開發中的修改在主資料夾進行，不影響 Release 版本。

**複製時排除以下項目：**
- `node_modules/` — 使用者首次執行時自動安裝
- `data/mops.db` — 個人設定與通知歷史，各自獨立
- `.env` — 含 API Key 等機密，不能公開

**注意**：`data/stock_ref.db` 為公開資料，**應隨版本一起發布**，使用者才有股票名稱對照表。
